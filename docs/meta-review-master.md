## A. threads_content_publish（再申請）

> **前回の却下理由**: スクリーンキャストが機能を実演していない
> **必須**: (1) アプリ内で投稿を作成・公開 (2) 同じ投稿がThreadsに表示されていること

### 申請テキスト

```
1. アプリの概要
当アプリ（ANALYCA）は、Threadsアカウントのコンテンツ投稿と分析を管理するためのWebダッシュボードです。Threads APIを利用して、ダッシュボードから直接Threadsへの即時投稿・予約投稿を行います。

2. 使用目的
即時投稿：ダッシュボードから直接Threadsへテキスト投稿する
投稿予約：投稿テキストと投稿日時を設定し、指定日時にThreads APIを通じて自動投稿する
コメント追加：メイン投稿の直後に、補足コメント（返信）を自動で追加する

3. ユーザーへの付加価値
投稿スケジュールをカレンダーで一元管理でき、投稿漏れを防止
投稿の事前準備により、コンテンツ品質を担保
手動でThreadsアプリを開いて投稿する手間を削減

4. threads_content_publish が必須な理由
当権限なしでは、Threads APIを通じたコンテンツの投稿ができません。ダッシュボードからの予約投稿・即時投稿はこの権限に完全に依存しています。投稿コンテンツはユーザーが作成したもののみで、ユーザーの明示的な操作によってのみAPIに送信されます。
```

### スクリーンキャスト撮影手順

**① ANALYCAにThreadsでログイン（20秒）**
- https://analyca.jp/login を開く
- 「Threadsでログイン」ボタンをクリック
- ポップアップが開く → Threads OAuth同意画面が表示される
- パーミッション一覧（threads_basic, threads_content_publish, threads_manage_insights, threads_manage_replies, threads_read_replies）が見えるようにゆっくり見せる
- 「許可」をクリック → ポップアップが閉じてダッシュボードへ

**② 予約投稿画面を開く（10秒）**
- Threadsタブが表示されている状態で「予約投稿」サブタブをクリック
- カレンダーと予約エディタが表示されるのを待つ

**③ カレンダーで既存の予約を見せる（15秒）**
- カレンダー上に投稿件数が表示されていることをゆっくり見せる
- 日付をクリックして予約一覧を表示
- 「予約済み」「投稿完了」のステータスが見えるようにする

**④ 即時投稿をデモする（40秒）** ← これが最重要
- 右側の「予約エディタ」にカーソルを合わせる
- メイン投稿テキストを**手入力**（例: 「ANALYCAからの投稿テストです」）
- コメント1を**手入力**（例: 「補足情報はこちら」）
- **「今すぐ投稿」ボタンをクリック**
- 投稿完了の表示を確認（成功メッセージが出る）

**⑤ Threadsで投稿を確認する（20秒）** ← これも最重要
- 別タブで https://threads.net を開く
- 自分のプロフィール（kudooo_ai）に移動
- **先ほどの投稿がThreads上に表示されていることをゆっくり見せる**
- コメント（返信）も表示されていることを確認

**⑥ 録画停止 → MP4保存**

## B. threads_manage_replies（新規申請）

> ※ Aと同じ動画を使い回せる可能性が高い（投稿+コメントが同時に映るため）

### 申請テキスト

```
1. アプリの概要
当アプリ（ANALYCA）は、Threadsアカウントのコンテンツ投稿と分析を管理するためのWebダッシュボードです。Threads APIを利用して、投稿の作成と同時にコメント欄（返信）を自動で追加する機能を提供します。

2. 使用目的
コメント追加：メイン投稿の直後に、補足コメント（返信）を自動で追加する。1つの投稿に対して最大2つのコメントを連続で投稿できる
返信管理：投稿したコメント（返信）の表示状態を管理する

3. ユーザーへの付加価値
メイン投稿とコメント欄の内容を事前に一括で準備でき、投稿時に自動でコメントが追加される
Threadsアプリ上で1つずつ手動で返信を投稿する手間を削減
コメント欄を活用した情報発信（続きはコメント欄へ、など）を効率化

4. threads_manage_replies が必須な理由
当権限なしでは、Threads APIを通じて投稿への返信（コメント）を作成できません。メイン投稿に対するコメント追加はANALYCAの主要機能であり、この権限に完全に依存しています。コメントの作成はユーザーの明示的な操作によってのみ行われ、自動的にスパムコメントを投稿するような使い方は一切しません。
```

### スクリーンキャスト撮影手順

> Aと別撮りする場合のみ。Aの動画でコメント追加が映っていればそちらを流用。

**② 投稿+コメント追加をデモする（50秒）**

- コメント1を手入力（例: 「続きはこちら。詳しく解説します。」）
- コメント2を手入力（例: 「まとめ：ポイントは3つです。」）
- 「今すぐ投稿」ボタンをクリック
- 投稿完了の表示を確認

**③ Threadsで返信を確認する（25秒）**
- https://threads.net でkudooo_aiのプロフィールへ
- 先ほどの投稿を開く
- メイン投稿の下にコメント1とコメント2が返信として表示されていることを確認

**④ 録画停止 → MP4保存**

## C. threads_read_replies（新規申請）

### 申請テキスト

```
1. アプリの概要
当アプリ（ANALYCA）は、Threadsアカウントの投稿パフォーマンスを分析するSaaS型ダッシュボードです。ユーザーがOAuth認証でThreadsアカウントを連携すると、投稿データ・インサイト・自身のリプライ（コメント欄）のインプレッション数を自動取得し、エンゲージメント分析を行います。

2. アプリのどの機能にこのアクセス許可が必要か
Threadsでは、1つのメイン投稿に対してユーザー自身が連続でリプライ（返信）を追加し、長文コンテンツを「コメント欄1→コメント欄2→コメント欄3」という形式で投稿するのが一般的な運用方法です。

threads_read_repliesは、この「ユーザー自身が投稿したリプライ（コメント欄）」のインプレッション数（views）を取得するために必要です。具体的には以下の機能で使用します：

- コメント欄遷移率分析：メイン投稿のviews → コメント欄1のviews → コメント欄2のviews → コメント欄3のviewsを取得し、各段階での閲覧数の推移をファネル形式で可視化する
- コメント別パフォーマンス比較：各コメント欄の閲覧数を比較し、どのコメントで読者の離脱が発生しているかを特定する

3. アクセス許可によってアプリの機能がどのように強化されるか
threads_read_repliesがなければ、メイン投稿のインサイト（views, likes等）は取得できますが、ユーザー自身のリプライ（コメント欄）のインプレッション数は取得できません。

この権限により、メイン投稿だけでなく各コメント欄の閲覧数まで含めた包括的な投稿分析が可能になります。これはANALYCAの主要な分析機能であり、この権限なしでは実現できません。

4. エンドユーザーの体験がどのように強化されるか
- コメント欄の何番目で読者が離脱しているかを数値で把握でき、投稿構成の改善に直結する
- 「メイン投稿は読まれているが、コメント欄2以降の閲覧数が急落している」といった具体的なインサイトが得られる
- 手動でThreadsアプリ上の各リプライのインプレッションを1つずつ確認する手間を削減し、ダッシュボードで一括分析できる
- データに基づいた投稿戦略の最適化が可能になる

5. 補足
- 取得対象はユーザー自身が投稿したリプライ（コメント欄）のみです。他のユーザーからのリプライは取得しません
- データの読み取りのみで、リプライの作成・変更・削除は行いません（作成にはthreads_manage_repliesを使用）
- 取得したデータはユーザー自身のダッシュボードにのみ表示され、第三者への公開や共有は一切行いません
```

### スクリーンキャスト撮影手順

**① ANALYCAダッシュボードを開く（10秒）**
- ANALYCAダッシュボード → Threadsタブ → 「分析」サブタブ

**② 投稿一覧でコメント数が表示されていることを見せる（15秒）**
- 投稿一覧をスクロールし、各投稿にコメント数が表示されていることを見せる

**③ コメント欄遷移率の分析を見せる（40秒）** ← 最重要
- コメント欄がある投稿をクリックして詳細を開く
- 「メイン → コメント欄1 → コメント欄2」の遷移率ファネルが表示されることを確認
- 各コメントの閲覧数（views）が表示されていることを見せる
- 遷移率のパーセンテージをゆっくり見せる

**④ Threadsで実際のコメントを確認する（20秒）**
- https://threads.net でkudooo_aiのプロフィールへ
- ダッシュボードで表示していた投稿を開く
- 実際のコメント（返信）がThreads上に存在していることを確認

**⑤ 録画停止 → MP4保存**
