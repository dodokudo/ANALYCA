# threads_content_publish アプリレビュー申請資料

## 1. 申請テキスト（Meta Developer Dashboardに貼り付け）

### 「このアクセス許可や機能の使用方法をお聞かせください」への回答

```
1. アプリの概要
当アプリ（AutoStudio）は、自社で運用しているThreadsアカウントのコンテンツ投稿を管理するための社内ツールです。Webブラウザ上のダッシュボードから、Threads APIを利用して自社アカウントへの予約投稿・即時投稿を行います。社内担当者のみがアクセスできます。

2. 使用目的
投稿予約：社内担当者がダッシュボード上で投稿テキストと投稿日時を設定し、指定日時にThreads APIを通じて自動投稿する
即時投稿：ダッシュボードから直接Threadsへ投稿する
コメント追加：メイン投稿の直後に、補足コメント（返信）を自動で追加する

3. ユーザー（社内担当者）への付加価値
投稿スケジュールをカレンダーで一元管理でき、投稿漏れを防止
投稿の事前準備と承認フローにより、コンテンツ品質を担保
手動でThreadsアプリを開いて投稿する手間を削減

4. threads_content_publish が必須な理由
当権限を付与しない限り、Threads APIを通じたコンテンツの投稿ができません。ダッシュボードからの予約投稿・即時投稿はこの権限に完全に依存しており、社内のThreads運用フローに不可欠です。社内限定利用のため、エンドユーザーへの提供・外部利用は一切行いません。投稿コンテンツは社内担当者が作成したもののみで、ユーザーの明示的な操作によってのみAPIに送信されます。
```

## 2. 動画撮影手順（工藤さん向け）

### 撮影前の準備
- Chromeで https://asto.jp にログインしておく
- 画面収録ソフトを起動（QuickTime Player → ファイル → 新規画面収録）
- 解像度: 1280x720以上
- 音声: 不要（Metaのレビュアーは音声を確認しない）
- 長さ: 1分30秒〜3分（過去の承認動画は約1分36秒〜1分54秒）
- 保存形式: MP4

### 撮影の流れ（この順番で操作）

**① Threads予約投稿画面を開く（10秒）**
- https://asto.jp/threads?range=7d&tab=schedule にアクセス
- 予約カレンダーと予約エディタが画面に表示されるのを待つ

**② カレンダーで既存の予約を見せる（15秒）**
- カレンダー上に投稿件数が表示されているのをゆっくり見せる（実際に使っている証拠）
- 日付をクリックして予約一覧を表示
- 「予約済み」「投稿完了」のステータスが見えるようにする

**③ 即時投稿をデモする（40秒）**
- 右側の「予約エディタ」にカーソルを合わせる
- メイン投稿テキストを手入力（例: 「AutoStudioからの投稿テストです」）
- コメント1を入力（例: 「補足情報」）
- 「今すぐ投稿」ボタンをクリック
- 投稿完了の表示を確認

**④ Threadsで投稿を確認する（20秒）**
- 別タブで https://threads.net を開く
- 自分のプロフィール（kudooo_ai）に移動
- 先ほどの投稿がThreads上に表示されていることを確認
- コメント（返信）も表示されていることを確認

**⑤ 録画停止**
- 画面収録を停止
- MP4で保存

### 撮影のポイント
- マウスの動きはゆっくり（レビュアーが追えるように）
- 各操作の間に1〜2秒の間を入れる
- テキストは実際にキーボードで入力する（コピペNG）
- **最重要**: Threadsアプリ上で投稿が実際に表示されている場面を必ず含める
