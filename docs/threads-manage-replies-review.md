# threads_manage_replies アプリレビュー申請資料

## 1. 申請テキスト（Meta Developer Dashboardに貼り付け）

### 「このアクセス許可や機能の使用方法をお聞かせください」への回答

```
1. アプリの概要
当アプリ（AutoStudio）は、Threadsアカウントのコンテンツ投稿を管理するためのWebダッシュボードです。Threads APIを利用して、投稿の作成と同時にコメント欄（返信）を自動で追加する機能を提供します。

2. 使用目的
コメント追加：メイン投稿の直後に、補足コメント（返信）を自動で追加する。1つの投稿に対して複数のコメントを連続で投稿できる
返信管理：投稿したコメント（返信）の表示状態を管理する

3. ユーザーへの付加価値
メイン投稿とコメント欄の内容を事前に一括で準備でき、投稿時に自動でコメントが追加される
Threadsアプリ上で1つずつ手動で返信を投稿する手間を削減
コメント欄を活用した情報発信（続きはコメント欄へ、など）を効率化

4. threads_manage_replies が必須な理由
当権限なしでは、Threads APIを通じて投稿への返信（コメント）を作成できません。メイン投稿に対するコメント追加はAutoStudioの主要機能であり、この権限に完全に依存しています。コメントの作成はユーザーの明示的な操作によってのみ行われ、自動的にスパムコメントを投稿するような使い方は一切しません。
```

## 2. 動画撮影手順（工藤さん向け）

### 撮影前の準備
- Chromeで https://asto.jp にログインしておく
- 画面収録ソフトを起動（QuickTime Player → ファイル → 新規画面収録）
- 解像度: 1280x720以上
- 音声: 不要
- 長さ: 1分30秒〜2分30秒
- 保存形式: MP4

### 撮影の流れ（この順番で操作）

**① Threads投稿画面を開く（10秒）**
- https://asto.jp/threads?range=7d&tab=schedule にアクセス
- 予約エディタが表示されるのを待つ

**② 投稿+コメント追加をデモする（50秒）**
- メイン投稿テキストを手入力（例: 「AutoStudioからの返信管理テストです」）
- 「コメントを追加」でコメント1を入力（例: 「続きはこちら。詳しく解説します。」）
- さらに「コメントを追加」でコメント2を入力（例: 「まとめ：〜のポイントは3つです。」）
- 「今すぐ投稿」ボタンをクリック
- 投稿完了の表示を確認

**③ Threadsで返信を確認する（25秒）**
- 別タブで https://threads.net を開く
- 自分のプロフィール（kudooo_ai）に移動
- 先ほどの投稿を開く
- メイン投稿の下にコメント1とコメント2が返信として表示されていることを確認

**④ 録画停止**
- 画面収録を停止
- MP4で保存

### 撮影のポイント
- マウスの動きはゆっくり（レビュアーが追えるように）
- 各操作の間に1〜2秒の間を入れる
- テキストは実際にキーボードで入力する（コピペNG）
- **最重要**: Threads上でコメント（返信）が実際に表示されている場面を必ず含める

## 3. 申請手順

1. Meta Developer Dashboard → ThreadsDataCollector → ユースケース → Threads APIにアクセス
2. `threads_manage_replies` の「アプリレビューに追加」ボタンをクリック
3. 確認 → アプリレビューで下書きが作成される
4. 「編集」をクリック
5. 「このアクセス許可や機能の使用方法をお聞かせください」欄に上記テキストを貼り付け
6. 動画をアップロード（MP4, 4GB以下）
7. 「送信」をクリック
