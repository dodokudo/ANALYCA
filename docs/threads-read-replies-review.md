# threads_read_replies アプリレビュー申請資料

## 1. 申請テキスト（Meta Developer Dashboardに貼り付け）

### 「このアクセス許可や機能の使用方法をお聞かせください」への回答

```
1. アプリの概要
当アプリ（ANALYCA）は、Threadsアカウントの投稿パフォーマンスを分析するSaaS型ダッシュボードです。ユーザーがOAuth認証でThreadsアカウントを連携すると、投稿データ・インサイト・コメントを取得し、エンゲージメント分析やコメント欄遷移率の可視化を自動で行います。

2. 使用目的
コメント欄遷移分析：各投稿のコメント（返信）を取得し、メイン投稿→コメント欄1→コメント欄2...という閲覧数の遷移率をファネル形式で可視化する
エンゲージメント分析：コメント数を含む総合的なエンゲージメント率を算出し、投稿ごとのパフォーマンスを比較する
コメント閲覧数の取得：各コメントのviews数を取得し、どのコメントが最も読まれているかを分析する

3. ユーザーへの付加価値
Threads投稿のコメント欄の「遷移率」（メイン投稿からコメント欄への離脱率）を数値化できる唯一のツール
コメント欄の構成（何番目のコメントで離脱が起きるか）を可視化し、投稿戦略の改善に直結する
手動でコメントの閲覧数を確認する手間を削減し、データドリブンな運用を実現

4. threads_read_replies が必須な理由
当権限なしでは、Threads APIを通じて投稿への返信（コメント）データを取得できません。コメント欄遷移率の分析はANALYCAの主要機能であり、この権限に完全に依存しています。取得したコメントデータはユーザー自身のダッシュボードにのみ表示され、第三者への公開や共有は一切行いません。コメントの読み取りのみで、コメントの作成・変更・削除は行いません。
```

## 2. 動画撮影手順（工藤さん向け）

### 撮影前の準備
- Chromeで https://analyca.jp にログインしておく（kudooo_aiのダッシュボードが表示される状態）
- 画面収録ソフトを起動（QuickTime Player → ファイル → 新規画面収録）
- 解像度: 1280x720以上
- 音声: 不要
- 長さ: 1分30秒〜2分30秒
- 保存形式: MP4

### 撮影の流れ（この順番で操作）

**① ANALYCAダッシュボードを開く（10秒）**
- https://analyca.jp/（自分のユーザーID）?tab=threads にアクセス
- Threads分析ダッシュボードが表示されるのを待つ

**② 投稿一覧でコメント数が表示されていることを見せる（15秒）**
- 投稿一覧をスクロールし、各投稿にコメント数が表示されていることを見せる
- コメント欄がある投稿をゆっくり見せる

**③ コメント欄遷移率の分析を見せる（40秒）**
- コメント欄がある投稿をクリックして詳細を開く
- 「メイン → コメント欄1 → コメント欄2」の遷移率ファネルが表示されることを確認
- 各コメントの閲覧数（views）が表示されていることを見せる
- 遷移率のパーセンテージをゆっくり見せる

**④ Threadsで実際のコメントを確認する（20秒）**
- 別タブで https://threads.net を開く
- 自分のプロフィール（kudooo_ai）に移動
- 先ほどダッシュボードで表示していた投稿を開く
- 実際のコメント（返信）がThreads上に存在していることを確認

**⑤ 録画停止**
- 画面収録を停止
- MP4で保存

### 撮影のポイント
- マウスの動きはゆっくり（レビュアーが追えるように）
- 各操作の間に1〜2秒の間を入れる
- **最重要**: ダッシュボード上のコメントデータとThreads上の実際のコメントが一致していることを見せる

## 3. 申請手順

1. Meta Developer Dashboard → ThreadsDataCollector → ユースケース → Threads APIにアクセス
2. `threads_read_replies` の「アプリレビューに追加」ボタンをクリック
3. 確認 → アプリレビューで下書きが作成される
4. 「編集」をクリック
5. 「このアクセス許可や機能の使用方法をお聞かせください」欄に上記テキストを貼り付け
6. 動画をアップロード（MP4, 4GB以下）
7. 「送信」をクリック
